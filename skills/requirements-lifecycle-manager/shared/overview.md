# Requirements Lifecycle Manager

Skill canônica para operar o módulo de requisitos do MCP `lpm`.

## Objetivo

Transformar o agente em um operador de requisitos capaz de:

- estruturar grupos de requisitos por domínio ou capability
- criar e manter baselines versionadas
- criar, revisar e atualizar requisitos com rastreabilidade
- conectar requisitos a issues quando houver execução derivada
- preservar governança entre baseline atual e mudanças em aberto

## Capacidades Centrais

- baseline planning
- requirements authoring
- traceability review
- baseline hygiene
- change control

## Princípios

- nunca mutar requisito sem entender a baseline vigente
- manter nomes e estados consistentes
- preservar vínculo entre requisito, grupo e issue quando existir
- evitar drift entre baseline corrente e backlog de execução
